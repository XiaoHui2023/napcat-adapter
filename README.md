# NapCat Adapter

## 设计特性

### 统一消息

模块把 NapCat 收到的私聊与群聊消息整理成统一消息模型，交给上游网关消费。读者只需要关心消息来自哪个会话、由谁发送、包含哪些消息段。

### 双向投递

模块同时支持从 NapCat 收消息和向 NapCat 发消息。来自网关的消息会按会话类型投递到对应目标，来自 NapCat 的消息会转成网关认识的结构。

### 会话识别

私聊使用发送者作为会话标识，群聊使用群作为会话标识。这样上游可以用同一套字段处理不同来源的对话。

### 消息边界

文本、提及、图片、位置、视频等常见消息段会保留为结构化数据。暂时无法投递给上游的消息段会被跳过，避免因为单个特殊内容阻断整条链路。

### 身份缓存

连接建立后会读取当前机器人基础信息，并随消息一起传递。上游可以据此识别机器人自身，处理群聊里与机器人相关的提及语义。

## 配置

### `napcat_ws_url`

| 项 | 说明 |
|---|---|
| 用途 | NapCat 对外提供的连接地址，填写为与 NapCat 侧一致的地址 |
| 示例 | `ws://127.0.0.1:3001` |

### `napcat_token`

| 项 | 说明 |
|---|---|
| 用途 | NapCat 访问令牌，填写为与 NapCat 侧一致的令牌；未启用令牌时可留空 |
| 示例 | `your-token` |

### `server_port`

| 项 | 说明 |
|---|---|
| 用途 | 网关服务端口，需与对接方填写的端口保持一致 |
| 默认值 | `8080` |

### `log_level`

| 项 | 说明 |
|---|---|
| 用途 | 日志级别，用于控制运行日志的详细程度 |
| 默认值 | `info` |
| 可选值 | `info`、`debug`、`warning`、`error`、`critical` |

### `log_dir`

| 项 | 说明 |
|---|---|
| 用途 | 日志目录；不填写时只输出到控制台 |
| 默认值 | 空 |

## API

| 名称 | 用途 |
|---|---|
| `AtSegment` | 表示消息中的提及内容，包含目标账号与展示名称 |
| `BaseSegment` | 所有消息段的基础结构，保留消息段类型与原始数据 |
| `Bot` | NapCat 适配器总类，负责收发消息并暴露回调注册能力 |
| `BotMessage` | 模块内部与网关之间传递的统一消息结构 |
| `FaceSegment` | 表示 QQ 表情消息段；适合识别内容类型，未必能继续转给上游 |
| `ForwardSegment` | 表示合并转发消息段 |
| `ImageSegment` | 表示图片消息段，保留文件、地址与摘要信息 |
| `JsonSegment` | 表示 JSON 消息段 |
| `LocationSegment` | 表示位置消息段，保留经纬度与展示文本 |
| `MessageType` | 表示消息来源类型，区分私聊与群聊 |
| `MfaceSegment` | 表示商城表情消息段，保留表情包与摘要信息 |
| `ReplySegment` | 表示回复消息段，保留被回复消息标识 |
| `TextSegment` | 表示纯文本消息段 |
| `VideoSegment` | 表示视频消息段，保留文件与地址信息 |
| `bot_to_onebot` | 把 NapCat 侧统一消息转换为网关可接收的消息载荷 |
| `onebot_to_bot` | 把网关消息载荷转换为 NapCat 侧统一消息 |
